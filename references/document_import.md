# 医学 Quiz 文档导入指南

> 本文档描述如何将外部文档（PDF/Word/PowerPoint）导入医学 Quiz 题库系统。

---

## 1. 支持格式

| 格式 | 扩展名 | 提取工具 | 说明 |
|------|--------|---------|------|
| Word 文档 | `.docx` | `docx` 技能 / python-docx | 支持正文段落、表格、加粗红色等格式 |
| PowerPoint | `.pptx` | `pptx` 技能 | 支持幻灯片标题、正文、备注栏 |
| PDF 文档 | `.pdf` | `pdf` 技能 | 支持文本提取，扫描版无法提取 |

## 3. 导入流程（详细步骤）

### 步骤 1：识别文件格式

```
输入：文件路径
处理：
  - 检查文件扩展名（大写/小写均支持）
  - 验证文件魔数（可选，防止扩展名伪造）
  - 返回格式类型：PDF | DOCX | PPTX | UNKNOWN
```

**支持列表**：
- `.pdf` → PDF
- `.docx` / `.doc` → Word（仅 docx 支持）
- `.pptx` → PowerPoint

### 步骤 2：调用对应技能提取文本

| 格式 | 调用技能 | 主要接口 |
|------|---------|---------|
| PDF | `pdf` | `pdf 技能` → 提取文本/表格/图片描述 |
| DOCX | `docx` | `docx 技能` → 提取段落/表格/格式 |
| PPTX | `pptx` | `pptx 技能` → 提取幻灯片/备注 |

### 步骤 3：文本清洗

去除以下噪声元素：

| 噪声类型 | 示例 | 处理方式 |
|---------|------|---------|
| 页眉页脚 | "第 1 页 / 共 10 页" | 正则匹配并删除 |
| 页码 | "1", "2", "3"（独立一行） | 删除独立数字行 |
| 空行 | 连续两个以上换行 | 压缩为单个换行 |
| 特殊字符 | `\x00`, `\t`（多余） | 替换为空格 |
| 超链接 | `http://xxx` | 删除 URL，保留链接文字 |

**清洗规则**：
```
1. 替换 \r\n 为 \n（Windows → Unix 换行）
2. 删除页眉页脚（匹配 "第 X 页" 等模式）
3. 删除独立数字行（整行仅为数字）
4. 压缩连续空行为单个换行
5. 去除首尾空白字符
```

### 步骤 4：知识点分段

按以下规则将文本切分为独立知识点：

| 切分依据 | 优先级 | 说明 |
|---------|------|------|
| 标题级别（##/###） | P0 | Markdown 标题或 Word 大纲 |
| 空行段落 | P1 | 独立段落用空行分隔 |
| 数字编号 | P2 | "1." "2." "（1）" 等 |
| 表格 | P3 | 每个表格作为一个知识点 |

**输出格式**：
```json
{
  "segments": [
    {
      "id": 1,
      "type": "heading",
      "level": 1,
      "title": "细胞的基本功能",
      "content": "..."
    },
    {
      "id": 2,
      "type": "paragraph",
      "title": null,
      "content": "物质转运包括：单纯扩散、易化扩散、主动转运..."
    },
    {
      "id": 3,
      "type": "table",
      "title": "转运方式对比",
      "headers": ["类型", "方向", "能量", "示例"],
      "rows": [["单纯扩散", "...", "...", "..."]]
    }
  ]
}
```

### 步骤 5：关键信息标注

自动识别并标注以下关键词标签：

| 标签 | 关键词模式 | 用途 |
|------|-----------|------|
| `【定义】` | "是指"、"称为"、"叫做" | 标记定义性语句 |
| `【鉴别】` | "鉴别"、"区别"、"不同点" | 标记鉴别诊断内容 |
| `【首选】` | "首选"、"首先"、"首选药" | 标记首选治疗/检查 |
| `【最常见】` | "最常见"、"最常见于"、"主要" | 标记流行病学数据 |
| `【特征性】` | "特征性"、"典型"、"特点" | 标记特征性表现 |
| `【时间窗】` | "小时内"、"分钟内"、"时间窗" | 标记时间敏感性内容 |
| `【数值】` | 数字 + 单位（mg/kg、mmol/L 等） | 标记数值型知识点 |

**标注示例**：
```
原文：洋地黄是治疗慢性心力衰竭的首先药物。
标注后：洋地黄是治疗慢性心力衰竭的【首选】药物。
```

### 步骤 6：存入题库索引

```
题库索引结构：
medical_quiz_library/
├── index.json              # 总索引
├── sources/                # 原始文档
│   ├── doc_001.pdf
│   ├── doc_002.docx
│   └── ...
├── segments/               # 切分段落
│   ├── seg_001.md
│   ├── seg_002.md
│   └── ...
└── keywords/               # 关键词索引
    └── keywords.json
```

**index.json 格式**：
```json
{
  "version": "1.0",
  "last_updated": "2026-04-15",
  "documents": [
    {
      "id": "doc_001",
      "filename": "生理学讲义.pdf",
      "format": "pdf",
      "imported_at": "2026-04-15T10:30:00",
      "segment_count": 45,
      "subject": "生理学"
    }
  ],
  "segments": [
    {
      "id": "seg_001",
      "doc_id": "doc_001",
      "type": "heading",
      "title": "细胞的基本功能",
      "tags": ["生理学"],
      "keywords": ["细胞", "功能"]
    }
  ]
}
```

## 4. 各格式提取规则

### 4.1 PDF 提取

**工具调用**：`pdf` 技能

**提取策略**：
```
1. 保留章节标题结构（h1/h2/h3）
2. 提取表格内容（转为 Markdown 表格）
3. 识别关键词行（"最常见"、"首选"、"特征性"等）
4. 跳过装饰元素（页眉/页脚/页码/水印）
```

**表格转换规则**：
```
原 PDF 表格：
┌─────────────┬──────────┐
│ 转运方式    │ 举例     │
├─────────────┼──────────┤
│ 单纯扩散    │ O2、CO2  │
└─────────────┴──────────┘

↓ 转为 Markdown 表格

| 转运方式 | 举例 |
|---------|------|
| 单纯扩散 | O2、CO2 |
```

**扫描版 PDF 处理**：
```
检测结果：扫描版 PDF（无可提取文字）
处理方式：
  1. 提示用户："此 PDF 为扫描版，无法直接提取文字"
  2. 提供建议："可使用 OCR 功能将扫描版转为可搜索 PDF"
  3. 不中断流程，记录失败
```

**特殊标注**：
```markdown
> ⚠️ 注意：以下内容来自扫描版 PDF，已人工确认文字准确性
```

### 4.2 Word 提取

**工具调用**：`docx` 技能

**提取内容**：
| 内容类型 | 提取方式 | 处理 |
|---------|---------|------|
| 正文段落 | 逐段提取 | 去除格式代码 |
| 表格 | 按单元格提取 | 转为 Markdown 表格 |
| 加粗文字 | 标记为 `**加粗内容**` | 表示重点 |
| 红色文字 | 保留原色或标记 `<红色>` | 表示重要知识点 |
| SmartArt | 提取为文字描述 | 注明"此为 SmartArt 图形内容" |
| 图片 | 提取 alt 文字 | 无 alt 则标记为"[图片内容]" |

**格式保留示例**：
```
原文：
心肌细胞具有**自律性**、**兴奋性**、**传导性**和**收缩性**。

提取后：
心肌细胞具有**【重点】自律性**、**【重点】兴奋性**、**传导性**和**收缩性**。

红色文字：
洋地黄是治疗心力衰竭的<红色>首选药物</红色>。
```

### 4.3 PowerPoint 提取

**工具调用**：`pptx` 技能

**提取内容**：
| 幻灯片元素 | 提取内容 | 处理 |
|-----------|---------|------|
| 幻灯片标题 | 第一行文本 | 保留为标题 |
| 幻灯片正文 | 要点列表 | 逐条提取 |
| 备注栏 | 讲者备注 | 标注 `【备注】` |
| 图表 | alt 文字或描述 | 转为文字说明 |

**幻灯片结构**：
```
=== 第 3 页：心力衰竭的临床表现 ===
【备注】此处应重点讲解左心衰和右心衰的鉴别点

要点：
- 左心衰：肺循环淤血（呼吸困难、咳嗽咯血）
- 右心衰：体循环淤血（颈静脉怒张、肝肿大、下肢水肿）
- 全心衰：上述两者同时存在
```

**忽略元素**：
- 背景模板文字
- 装饰性图形
- 动画标记
- 幻灯片编号

## 5. 错误处理

### 5.1 错误类型与处理方式

| 错误类型 | 检测条件 | 处理方式 | 用户提示 |
|---------|---------|---------|---------|
| 文件不存在 | `os.path.exists() == False` | 中断，报告错误 | "❌ 文件不存在：`[路径]`" |
| 格式不支持 | 扩展名非 pdf/docx/pptx | 跳过，报告警告 | "⚠️ 格式不支持：`[文件名]`，已跳过" |
| 提取失败 | 技能返回异常/空内容 | 记录失败，继续下一个 | "❌ 提取失败：`[文件名]`，原因：[错误信息]" |
| 文本为空 | 提取内容长度 == 0 | 记录为空，报告 | "⚠️ 提取内容为空：`[文件名]`，已跳过" |
| 文件损坏 | 打开文件时异常 | 跳过，报告 | "❌ 文件损坏，无法打开：`[文件名]`" |

### 5.2 批量导入错误汇总

```
用户：/导入文件夹 D:/医学资料/
    ↓
✅ 批量导入完成！
共发现：15 个文件
成功导入：12 个
跳过（格式不支持）：1 个
提取失败：1 个
内容为空：1 个

失败文件列表：
  - 损坏文档.docx（文件损坏）
  - 空文件.pdf（提取内容为空）
  - notes.txt（格式不支持）
```

### 5.3 异常恢复

```
导入过程中断时：
1. 保存已导入部分到索引
2. 记录中断位置
3. 下次导入时：
   - 检查已导入文件列表
   - 跳过已成功导入的文件
   - 从中断位置继续
```

## 7. 技术实现参考

### 7.1 核心模块结构

```
medical_quiz/
├── core/
│   ├── document_loader.py    # 文档加载器（支持 PDF/DOCX/PPTX）
│   ├── text_cleaner.py       # 文本清洗模块
│   ├── segmenter.py          # 知识点分段器
│   ├── keyword_tagger.py     # 关键词标注器
│   └── index_manager.py      # 题库索引管理
├── skills/
│   └── document_import.md    # 本文档
└── references/
    └── 306_exam_outline.md   # 306 考纲
```

### 7.2 依赖工具

| 工具 | 版本 | 用途 |
|------|------|------|
| python-docx | ≥ 0.8.11 | Word 文档读取 |
| python-pptx | ≥ 0.6.21 | PowerPoint 读取 |
| PyPDF2 / pdfplumber | ≥ 2.0 | PDF 文本/表格提取 |
| 正则表达式（re） | 内置 | 文本清洗 |

### 7.3 性能考量

```
文档导入性能参考：
  - 100 页 PDF：约 10-30 秒
  - 50 页 Word：约 5-15 秒
  - 100 页 PPT：约 15-30 秒

批量导入建议：
  - 单次批量不超过 20 个文件
  - 大文件优先处理
  - 显示进度条提升用户体验
```
