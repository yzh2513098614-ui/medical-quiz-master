# Contributing to Medical Quiz Master

Thank you for your interest in contributing to Medical Quiz Master! This document provides guidelines and instructions for participating in this project.

## How to Contribute

### 1. Fork the Repository

Click the "Fork" button on the GitHub page to create your own copy of the repository.

### 2. Clone Your Fork

```bash
git clone https://github.com/your-username/medical-quiz-master.git
cd medical-quiz-master
```

### 3. Create a Feature Branch

```bash
git checkout -b feature/your-feature-name
```

### 4. Make Your Changes

- Write your code following the coding standards below
- Add or update tests as needed
- Update documentation if applicable

### 5. Commit Your Changes

```bash
git add .
git commit -m "Add: brief description of your changes"
```

### 6. Push to Your Fork

```bash
git push origin feature/your-feature-name
```

### 7. Create a Pull Request

Go to the original repository and click "New Pull Request". Describe your changes clearly and why they would be valuable.

## Coding Standards

### General Guidelines

- Write code that is clear, readable, and well-organized
- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions focused and single-purpose

### Documentation

- Update README.md if you add new features
- Add docstrings to new functions
- Keep documentation in Chinese (consistent with the project)

### Testing

- Test your changes before submitting
- Ensure existing tests still pass
- Add tests for new functionality

## Types of Contributions

### Bug Reports

- Describe the bug in detail
- Include steps to reproduce
- Note your environment (OS, version, etc.)

### Feature Suggestions

- Explain the feature and its benefit
- Provide use case examples
- Consider backward compatibility

### Documentation Improvements

- Fix typos or unclear wording
- Add missing information
- Improve examples

### Code Contributions

- New question banks
- Bug fixes
- Performance improvements
- New features

## Questions?

If you have questions, feel free to open an issue for discussion.
